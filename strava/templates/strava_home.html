Strava Project

Screen Shots

Motivation

Strava is a popular social fitness platform used by athletes to track and analyze activities. I bought a GPS watch several years ago to track my running times where activities are uploaded to the Strava server, which can be accessed through API calls. In order to create custom dashboards for my data analysis, I developed these pages along with automated scripts that download new activities nightly.

Pages

Sync

This page is necessary for downloading activities on the same day. It redirects the user to Strava's website for OAuth2 authentication. Currently, it is restricted to those with admin privileges.

Activities

This page lists all activities with filtering capability. The hyperlinks open pages for the analysis of each specific activity.

Analysis

This page is for historical and accumulative analysis of many activities.

Implemented

  • API Interface
    • User initiated: OAuth2 redirect authentication
    • Nightly download: Automate OAuth2 login with headless Python/Selenium webform filling.
  • Database Design
    • Reduce storage size by computing 100-meter splits from activity streams.
    • Create helper tables for faster analysis (for example, aggregating data for similar activity routes).
  • Data Visualization
    • Mapbox API for latitude/longitude maps
    • Plotly.js library for plots and charts
    • JavaScript for interactivity

Future Work

  • Add analysis for matched (similar) activities
  • Allow users to name/save specific routes
